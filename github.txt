1.  git init  
Команда ініціалізує новий репозиторій Git в поточній директорії

2.  git add .
Додає всі зміни (нові, відредаговані та видалені файли) у поточній директорії до індексу Git

3.  git commit -m "first"
Команда створює коміт, який зберігає зміни в репозиторії Git

4.  git remote add origin https://github.com/USER_NAME/NAME_REPOSITORY.git
Команда додає віддалений репозиторій, з яким буде синхронізуватися ваш локальний репозиторій.
Ця команда "пов'язує" ваш локальний репозиторій з репозиторієм на GitHub, дозволяючи відправляти зміни на GitHub і отримувати їх назад

origin - це ім'я для вашого віддаленого репозиторію. Це стандартне ім'я, яке використовують для позначення основного віддаленого репозиторію.


5.  git push -u origin main
Команда відправляє ваші локальні зміни на віддалений репозиторій.

Параметр -u встановлює віддалену гілку як "за замовчуванням" для цієї локальної гілки. Це означає, що в майбутньому ви зможете використовувати команду git push без вказування гілки або віддаленого репозиторію, і Git буде знати, на який саме репозиторій і гілку потрібно відправляти зміни.
